# Invoice Troubleshooting

## Invoice lifecycle

```
draft → finalized → paid
                  → failed (retries) → paid
                                     → failed (exhausted → customer suspended)
                  → refunded
```

- **Draft**: Generated by billing run, not yet charged. Can be modified
- **Finalized**: Locked, ready for payment. Cannot be modified
- **Paid**: Successfully charged via Stripe
- **Failed**: Stripe charge failed. Retries managed by Stripe Smart Retries
- **Refunded**: Charge was reversed via Stripe

## Generating a manual invoice

### Via admin panel
1. Navigate to `/admin/customers/<id>` → Invoices tab
2. Click "Generate Invoice" → select month
3. Invoice is created in `draft` status

### Via admin API
```bash
curl -X POST https://api.flapjack.foo/admin/tenants/<id>/invoices \
  -H "X-Admin-Key: $ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"month": "2026-02"}'
```

## Finalizing a draft invoice

### Via admin panel
1. Navigate to customer detail → Invoices tab
2. Click "Finalize" on the draft invoice

### Via admin API
```bash
curl -X POST https://api.flapjack.foo/admin/invoices/<id>/finalize \
  -H "X-Admin-Key: $ADMIN_KEY"
```

## Running batch billing

Batch billing generates invoices for all active customers for a given month.

### Via admin panel
1. Navigate to `/admin/billing`
2. Select month from the month picker
3. Click "Run Billing" → confirm in the warning modal

### Via admin API
```bash
curl -X POST https://api.flapjack.foo/admin/billing/run \
  -H "X-Admin-Key: $ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"month": "2026-02"}'
```

## Investigating a failed payment

1. **Check Stripe dashboard**: Search for the customer or invoice ID. Look at payment attempts, decline codes, and retry schedule
2. **Check webhook events**: `GET /admin/alerts` — search for payment-related alerts with the invoice ID
3. **Common decline codes**:
   - `card_declined`: Generic decline. Customer needs to contact their bank
   - `insufficient_funds`: Customer needs to add funds or use a different card
   - `expired_card`: Customer needs to update their payment method
   - `authentication_required`: 3D Secure required. Customer needs to complete authentication

## Bulk finalization

Use `/admin/billing` → "Bulk Finalize" button to finalize all draft invoices at once. This is typically done after reviewing the batch billing run output.
