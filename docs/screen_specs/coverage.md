# Screen Spec Coverage

This map connects critical routes to target-behavior specs and automated tests.
Every critical route below has a dedicated spec. Criterion-level mapping lives in
each spec's `Automated Coverage` and `Current Implementation Gaps` sections so
this file stays a route-level index instead of a duplicated test matrix.

## Planned Standalone Operator Targets

The standalone operator contracts `docs/screen_specs/system.md`,
`docs/screen_specs/cluster.md`, and `docs/screen_specs/snapshots.md` are target
placements for the future standalone console surfaces owned by
`docs/design/console_unification_revised_plan.md` Decision R5. They map to
standalone `/system`, `/cluster`, and `/snapshots` behavior, while the current
managed fjcloud console has no repo-local route owner for these unimplemented
surfaces.

Current-behavior evidence comes from the external flapjack React dashboard and
HTTP owners rather than fjcloud-local promoted coverage:

- `docs/screen_specs/system.md`: `engine/dashboard/tests/specs/system.md`;
  `engine/dashboard/src/pages/System.tsx`; `engine/dashboard/src/hooks/useHealth.ts`;
  `engine/dashboard/src/hooks/useIndexes.ts`; `engine/flapjack-http/src/router.rs`;
  `engine/flapjack-http/src/handlers/health.rs`; `engine/flapjack-http/src/handlers/index.rs`.
- `docs/screen_specs/cluster.md`: `engine/dashboard/tests/specs/cluster.md`;
  `engine/dashboard/src/pages/Cluster.test.tsx`;
  `engine/dashboard/src/hooks/__tests__/useClusterStatus.test.ts`;
  `engine/dashboard/src/pages/Cluster.tsx`;
  `engine/dashboard/src/hooks/useClusterStatus.ts`; `engine/flapjack-http/src/router.rs`;
  `engine/flapjack-http/src/handlers/internal.rs`.
- `docs/screen_specs/snapshots.md`: `engine/dashboard/tests/specs/system.md` scenario
  `system-4: Snapshots tab`; `engine/dashboard/src/pages/System.tsx`;
  `engine/dashboard/src/pages/SystemTabSections.tsx`;
  `engine/dashboard/src/hooks/useSnapshots.ts`; `engine/dashboard/src/hooks/useIndexes.ts`;
  `engine/flapjack-http/src/router.rs`; `engine/flapjack-http/src/handlers/snapshot.rs`.
